# SEO-агенты для Claude Code

🇬🇧 [English version](README.md)

Набор из 10 специализированных SEO-сабагентов для [Claude Code](https://claude.com/claude-code). Каждый агент — узкий специалист: даёшь ему URL (или ключевое слово), он проводит свой аудит и возвращает структурированный отчёт с оценками. Агентов можно запускать по одному или параллельно — для полного аудита сайта.

## Установка

**Скачать всё разом** — [ZIP-архив](https://github.com/exelseve/seo-agents/archive/refs/heads/main.zip) (аккаунт GitHub не нужен), или через терминал:

```bash
git clone https://github.com/exelseve/seo-agents.git
cp seo-agents/sb-*.md ~/.claude/agents/
cp -r seo-agents/scripts ~/.claude/skills/seo/scripts   # вспомогательные скрипты (опционально, см. ниже)
```

Скопируй `.md`-файлы в одну из папок:

- `~/.claude/agents/` — будут доступны во всех проектах
- `<твой-проект>/.claude/agents/` — только в этом проекте

Дальше просто проси Claude Code: *«запусти sb-tech-audit и sb-ai-search по https://example.com параллельно»*.

## Вспомогательные скрипты

В папке `scripts/` лежат Python-хелперы, которые агенты ищут по пути `~/.claude/skills/seo/scripts/`: рендеринг страниц с JS, клиенты CrUX/PageSpeed API, скриншоты, drift-инструменты. Они **опциональны**: без них все агенты, кроме `sb-drift`, спокойно работают через `curl`/WebSearch. `sb-drift` — единственный, кому они обязательны (хранение снапшотов в SQLite).

Как подключить:

```bash
cp -r seo-agents/scripts ~/.claude/skills/seo/scripts
pip install -r ~/.claude/skills/seo/scripts/requirements.txt
playwright install chromium   # для render_page / capture_screenshot
```

| Скрипт | Что делает |
|---|---|
| `render_page.py` | Headless-рендер страницы через Chromium (Playwright): исполняет JavaScript и возвращает итоговый DOM, чистый основной текст и дату публикации. Разница между «что видит краулер без JS» и отрендеренной страницей — ключевой сигнал для нескольких агентов. |
| `fetch_page.py` | Обычный HTTP-фетчер с реалистичными заголовками, трассировкой редиректов и режимом Googlebot-UA для детекции динамического рендеринга. Все запросы проходят через SSRF-защиту. |
| `parse_html.py` | Извлекает SEO-критичные элементы из HTML (title, meta, canonical, заголовки, JSON-LD, OG-теги) в JSON; парсинг-бэкенд для drift-снапшотов. |
| `capture_screenshot.py` | Скриншоты через Playwright по вьюпортам (десктоп/мобайл) для визуального аудита. |
| `pagespeed_check.py` | Сводный отчёт PageSpeed Insights v5 + CrUX: лабораторные данные Lighthouse вместе с полевыми данными реальных пользователей. Нужен Google API-ключ. |
| `crux_history.py` | Тренды Core Web Vitals: до 25 недель истории из CrUX History API с классификацией «улучшается/деградирует» по каждой метрике. |
| `drift_baseline.py` | Снимает снапшот SEO-критичных элементов страницы в SQLite (`~/.cache/seo-agents/drift/`) как эталонное состояние. |
| `drift_compare.py` | Сравнивает живую страницу с эталоном: 17 правил в уровнях CRITICAL / WARNING / INFO. |
| `drift_history.py` | Таймлайн снапшотов и сравнений по URL. |
| `drift_report.py` | Рендерит результат сравнения в HTML-отчёт с цветовой кодировкой по серьёзности. |
| `url_safety.py` | Общая защита от SSRF/DNS-rebinding: блокирует запросы к приватным и зарезервированным IP. |
| `google_auth.py` | Загрузчик Google API-ключа (`~/.config/seo-agents/google-api.json` или переменная `GOOGLE_API_KEY`) для клиентов PageSpeed/CrUX. Инструкция по настройке: `python google_auth.py --setup`. |

## Агенты

| Агент | Что делает |
|---|---|
| `sb-tech-audit` | Технический SEO-аудит по 9 направлениям: краулинг, индексация, security-заголовки, гигиена URL, мобильная версия, риски Core Web Vitals, наличие микроразметки, модель рендеринга (SSR/CSR), IndexNow. Вердикт по каждой категории + балл 0–100 + приоритизированные исправления. |
| `sb-content-quality` | Оценивает контент как асессор Google: скоринг E-E-A-T (веса T30/E25/A25/E20), полнота раскрытия темы относительно минимумов по типу страницы, читабельность, естественность ключей, цитируемость для AI и детекция низкокачественного сгенерированного текста. |
| `sb-schema-audit` | Всё про Schema.org: находит существующую разметку (JSON-LD/Microdata/RDFa), валидирует по актуальным rich results, знает мёртвые типы (HowTo, SpecialAnnouncement…) и что FAQ-сниппеты остались только у гос- и медсайтов, генерирует готовый JSON-LD. |
| `sb-ai-search` | GEO (Generative Engine Optimization): будет ли AI-поиск *цитировать* страницу? Проверяет доступ AI-краулеров в robots.txt, `llms.txt`, цитируемость на уровне пассажей (самодостаточные ответы по 130–170 слов), сигналы авторитетности и готовность отдельно под Google AI Overviews, ChatGPT, Perplexity, Bing Copilot. |
| `sb-sxo` | Отвечает на вопрос «почему страница не ранжируется?» чтением выдачи «задом наперёд»: классифицирует топ-10, ловит несовпадение типа страницы (главный тихий убийца позиций), выводит user stories из сигналов SERP, оценивает страницу по 7 осям гэп-анализа и 4–7 персонам искателей. |
| `sb-performance` | Диагностика Core Web Vitals (LCP / INP / CLS, пороги 2026 года, правило 75-го перцентиля). Предпочитает полевые данные CrUX лабораторным, деградирует по цепочке PageSpeed API → Lighthouse → анализ исходника. Называет конкретное узкое место и ранжирует фиксы по эффекту. |
| `sb-sitemap` | Валидация XML-сайтмапов (формат, лимит 50k, мёртвые/редиректные/noindex-URL, честность lastmod) и генерация новых. Держит анти-doorway гейты: предупреждение при 30+ программных гео-страниц, жёсткий стоп при 50+. |
| `sb-visual` | Скриншотит страницы через Playwright в 4 вьюпортах и проверяет то, что реально видит пользователь: H1 и CTA до скролла, размеры тач-таргетов на мобиле, горизонтальный скролл, перекрытия и обрезанные элементы. |
| `sb-cluster` | Кластеризация ключей по доказательствам, а не по похожести текста: попарное измерение пересечения выдачи (общие URL в топ-10), классификация интента, проектирование hub-and-spoke архитектуры и полная матрица внутренней перелинковки. Выдаёт `cluster-plan.json` + человекочитаемое резюме. |
| `sb-drift` | Version control для on-page SEO: снимает снапшоты title/meta/canonical/robots/заголовков/схем/OG/CWV по URL в SQLite, сравнивает живую страницу с базлайном по 17 правилам в трёх уровнях (CRITICAL / WARNING / INFO) и направляет находки профильному агенту. |

## Типовые связки

- **Полный аудит**: `sb-tech-audit` + `sb-content-quality` + `sb-schema-audit` + `sb-ai-search` + `sb-sxo` параллельно, потом свести в один отчёт
- **«Упали позиции»**: `sb-drift` (что изменилось?) → профильный агент, на которого он укажет
- **Планирование контента**: `sb-cluster` → написание → `sb-content-quality` перед публикацией
- **Проверка после деплоя**: `sb-visual` + `sb-performance` + `sb-drift`

## Примечания

- Frontmatter (`model`, `maxTurns`, `tools`) настроен под каждого агента — правь под себя.
- Агенты никогда не дёргают пользовательские URL сырым `requests.get`: используют безопасный фетчер, если он есть, или обычный `curl`.
- Зашитые в промпты факты (пороги CWV, даты депрекаций схем, ссылки на QRG) актуальны на середину 2026 года.
